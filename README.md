## 🏗 Архитектура и паттерны

Проект реализован в рамках **компонентно-ориентированной парадигмы (ECS-подобный подход Unity)**. Логика строго разделена на независимые модули для соблюдения принципа единственной ответственности (**Single Responsibility Principle**):

> 💡 **Важно для чистоты кода:** 
> Разделение логики на спавнер и контроллер позволяет легко масштабировать проект. Если завтра джойстик заменится на тапы по экрану, код отслеживания маркеров (`ARTrackedImageSpawner`) останется абсолютно нетронутым.

*   **Модуль отслеживания (`ARTrackedImageSpawner.cs`)** — подписывается на системное событие `ARTrackableChangedEventArgs`, обрабатывает появление маркера в кадре и инстанцирует 3D-объект.
*   **Модуль контроллера (`CharizardController.cs`)** — отвечает за физическое перемещение (`Rigidbody`), считывание осей ввода джойстика и переключение состояний аниматора.

## 💻 Пошаговое развертывание проекта

### Этап 1: Настройка сцены и XR-компонентов
Стандартная камера удалена. Взамен развернута мобильная экосистема AR:
1. Добавлен `XR Origin (Mobile AR)` для перевода реальных координат гаджета в мировые координаты сцены.
2. Добавлен `AR Session` для контроля жизненного цикла дополненной реальности.
3. Создана библиотека `Reference Image Library` для хранения эталонного маркера.

### Этап 2: Конечный автомат анимаций (Animator)
Для плавного переключения движений модели настроен Animator Controller:
*   **Состояние Jump:** проигрывается один раз при появлении модели.
*   **Состояние Hover:** дефолтная анимация парения на месте.
*   **Состояние Fly:** анимация полета. Переход в нее завязан на логический параметр `fly_param`, который активируется только при ненулевом векторе движения джойстика.
