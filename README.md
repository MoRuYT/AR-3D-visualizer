## 1. Введение
В данном отчете описывается процесс выполнения практического задания по разработке приложения дополненной реальности (AR), способного распознавать 2D-изображения в реальном времени и проецировать на них интерактивные 3D-модели. Приложение разработано на базе игрового движка Unity с использованием технологий машинного зрения для трекинга маркеров.

## 2. Используемые технологии
Проект реализован с использованием следующего стека:
* **Unity Editor (LTS)** с модулями Android Build Support (Open JDK, Android SDK, NDK, IL2CPP, ARM64).
* **AR Foundation & Google ARCore XR Plugin** — для работы с дополненной реальностью на мобильных устройствах.
* **com.unity.cloud.gltfast** — для импорта 3D-моделей в открытом формате glTF.

## 3. Архитектура приложения
Разработка ведется в рамках компонентно-ориентированной парадигмы (Entity-Component-System концепция Unity). Для соблюдения принципа единственной ответственности логика строго разделена на независимые модули:
* **Модуль отслеживания (`ARTrackedImageSpawner.cs`)**: отвечает исключительно за подписку на системное событие обнаружения маркера (`ARTrackableChangedEventArgs`) и инстанцирование 3D-префаба в мировых координатах.
* **Модуль контроллера (`CharizardController.cs`)**: инкапсулирует физику (работа с компонентом `Rigidbody`), обработку ввода с наэкранного джойстика и управление состояниями конечного автомата анимаций (`Animator`).

Такое разделение позволяет масштабировать проект (например, при необходимости заменить UI-джойстик на другой метод ввода, код спавнера не потребует модификаций).

## 4. Этапы разработки

### 4.1. Настройка окружения
После установки движка были скорректированы параметры Player Settings: отключен Vulkan API, деактивирован Multi-threaded Rendering и выбрана архитектура ARM64 с бэкендом IL2CPP для корректной сборки под платформу Android. В настройках XR Plug-in Management в качестве провайдера активирован Google ARCore.

![Скриншот 1: Окно XR Plug-in Management в Unity](ссылка_на_твой_скриншот_1.jpg)
*(На скриншоте: демонстрация выбранного плагина ARCore)*

### 4.2. Подготовка AR-сцены
Стандартная игровая камера была заменена на структуру **XR Origin (Mobile AR)**, которая трансформирует реальные координаты устройства в мировые координаты Unity. Для инициализации жизненного цикла сессии добавлен компонент `AR Session`. К объекту XR Origin подключен скрипт `AR Tracked Image Manager`, в который передана библиотека эталонных изображений (`Reference Image Library`), содержащая таргет-маркер.

![Скриншот 2: Структура иерархии сцены (XR Origin, AR Session)](ссылка_на_твой_скриншот_2.jpg)
*(На скриншоте: настроенная иерархия объектов AR-сцены)*

### 4.3. Настройка 3D-модели и анимаций
В качестве визуализируемого объекта интегрирована 3D-модель с набором анимаций. Для нее был создан конечный автомат `Animator Controller` с тремя ключевыми состояниями: *Jump* (инициализация/старт), *Hover* (состояние покоя) и *Fly* (передвижение). Переходы между состояниями *Hover* и *Fly* управляются логическим параметром (bool parameter), который меняет значение в зависимости от ввода пользователя.

![Скриншот 3: Окно Animator с настроенными переходами](ссылка_на_твой_скриншот_3.jpg)
*(На скриншоте: логика переходов в Animator Controller)*

### 4.4. Реализация UI и управления
Для навигации объекта в AR-пространстве в слой Canvas интегрирован компонент Fixed Joystick. Скрипт контроллера считывает векторы `x` и `y` с джойстика, применяет к 3D-модели линейную скорость через `Rigidbody` и выполняет ротацию объекта в сторону движения.

## 5. Сборка и тестирование
Перед компиляцией в настройках рендерера был добавлен профиль `AR Background Renderer Feature`, необходимый для трансляции потока с камеры устройства на задний фон сцены. Сборка проекта (`Build and Run`) произведена на Android-устройство с включенным режимом отладки (USB Debugging).

## 6. Заключение
В ходе выполнения практики было спроектировано и разработано AR-приложение. Настроена интеграция с фреймворком ARCore, реализован стабильный трекинг 2D-маркеров, написаны модульные C#-скрипты, а также успешно реализовано физическое управление 3D-объектом в реальном пространстве с синхронизацией анимаций.
